## 🏗️ Architecture Overview

### 4-Tier Layered Architecture

```
┌─────────────────────────────┐
│    API Layer                │ ← REST endpoints, authentication
│    (Controllers)            │
├─────────────────────────────┤
│    Application Layer        │ ← Business logic, services, DTOs
│    (Services)               │
├─────────────────────────────┤
│    Domain Layer             │ ← Entities, domain rules, interfaces
│    (Entities)               │
├─────────────────────────────┤
│    Infrastructure Layer     │ ← Data access, repositories, EF Core
│    (Data Access)            │
├─────────────────────────────┤
│    Database                 │ ← SQL Server, 8 normalized entities
│    (SQL Server)             │
└─────────────────────────────┘
```

### Key Architecture Patterns

- **Clean Architecture**: Separation of concerns, testability
- **Repository Pattern**: Abstract data access
- **Dependency Injection**: Loose coupling
- **DTO Pattern**: Clean data transfer
- **RBAC**: Role-based access control
- **JWT Auth**: Stateless authentication
- **Audit Logging**: Track all changes
- **Multi-tenancy**: Organization isolation
