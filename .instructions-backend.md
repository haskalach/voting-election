- **Framework**: ASP.NET Core 10.0

- **Database**: SQL Server
- **Architecture**: Layered (API → Application → Domain → Infrastructure)
- **Current Phase**: 8 (AI Agent Configuration)

## Backend Architecture Overview

```
API Layer (Controllers)
    ↓ (calls)
Application Layer (DTOs, Services, Interfaces)
    ↓ (depends on)
Domain Layer (Entities, Repository Interfaces)
    ↓ (implemented by)
Infrastructure Layer (DbContext, Repository Implementations)
    ↓
Database
```

**Key Principle**: Each layer has clear responsibility. Never let lower layers depend on higher layers.

## Coding Standards for Agents

### 1. Async/Await (Non-negotiable)

✅ **ALWAYS use async**:

```csharp
// GOOD
public async Task<UserDto> GetUserAsync(int id)
{
    return await _userRepository.GetByIdAsync(id);
}

// GOOD - async void only for event handlers
public async void OnButtonClicked(object sender, EventArgs e)
{
    await HandleClickAsync();
}
```

❌ **NEVER sync over async**:

```csharp
// BAD - causes deadlocks
public UserDto GetUser(int id)
{
    return _userRepository.GetByIdAsync(id).Result;  // ❌ Deadlock
}

// BAD - blocking async call
public UserDto GetUser(int id)
{
    var user = _userRepository.GetByIdAsync(id).GetAwaiter().GetResult();  // ❌ Bad
    return user;
}
```

### 2. Dependency Injection (Constructor-based)

✅ **GOOD - Constructor injection**:

```csharp
public class OrganizationService : IOrganizationService
{
    private readonly IOrganizationRepository _orgRepo;
    private readonly IUnitOfWork _unitOfWork;

    public OrganizationService(
        IOrganizationRepository orgRepo,
        IUnitOfWork unitOfWork)
    {
        _orgRepo = orgRepo;
        _unitOfWork = unitOfWork;
    }
}
```

❌ **BAD - Service locator or property injection**:

```csharp
// BAD
public class OrganizationService
{
    [Inject]
    public IOrganizationRepository OrgRepo { get; set; }  // ❌ Property injection
}

// BAD - Service locator
public class OrganizationService
{
    public void DoSomething()
    {
        var repo = ServiceLocator.GetService<IOrganizationRepository>();  // ❌ Service locator
    }
}
```

### 3. Record Types for DTOs

✅ **GOOD - Immutable records**:

```csharp
public record LoginRequestDto(
    [Required][EmailAddress] string Email,
    [Required][MinLength(8)] string Password
);

public record OrganizationDto(
    int OrganizationId,
    string OrganizationName,
    int TotalEmployeeCount
);
```

❌ **BAD - Mutable classes**:

```csharp
// BAD
public class LoginRequestDto
{
    public string Email { get; set; }  // Mutable
    public string Password { get; set; }
}
```

### 4. Validation with DataAnnotations

✅ **GOOD - Declarative validation on DTOs**:

```csharp
public record CreateOrganizationDto(
    [Required][MaxLength(200)] string OrganizationName,
    [Required][EmailAddress] string ManagerEmail,
    [Required][MinLength(3)] string AdminFirstName,
    [Required][MinLength(3)] string AdminLastName
);
```

**API layer validates automatically**:

```csharp
[HttpPost]
public async Task<ActionResult<OrganizationDto>> Create([FromBody] CreateOrganizationDto dto)
{
    if (!ModelState.IsValid)
        return BadRequest(ModelState);  // ✅ Framework handles this

    var org = await _orgService.CreateAsync(dto, userId);
    return CreatedAtAction(nameof(GetById), new { id = org.OrganizationId }, org);
}
```

❌ **BAD - Manual validation in service**:

```csharp
public async Task<OrganizationDto> CreateAsync(CreateOrganizationDto dto)
{
    // ❌ Don't validate here - should follow ModelState check
    if (string.IsNullOrEmpty(dto.OrganizationName))
        throw new ArgumentException("Name required");
}
```

### 5. Error Handling & Exceptions

✅ **GOOD - Domain exceptions caught by API**:

```csharp
// Service throws domain exceptions
public async Task<OrganizationDto> DeleteAsync(int id)
{
    var org = await _orgRepo.GetByIdAsync(id)
        ?? throw new KeyNotFoundException("Organization not found");

    await _orgRepo.DeleteAsync(org);
    await _unitOfWork.SaveChangesAsync();
}

// Controller catches and translates to HTTP
[HttpDelete("{id}")]
public async Task<IActionResult> Delete(int id)
{
    try
    {
        await _orgService.DeleteAsync(id);
        return NoContent();
    }
    catch (KeyNotFoundException)
    {
        return NotFound();
    }
    catch (InvalidOperationException ex)
    {
        return BadRequest(new { message = ex.Message });
    }
}
```

❌ **BAD - Http status codes in services**:

```csharp
// ❌ Services shouldn't know about HTTP
public async Task<OrganizationDto> DeleteAsync(int id)
{
    try
    {
        // ...
    }
    catch
    {
        return StatusCode(500);  // ❌ Don't return HTTP codes
    }
}
```

### 6. XML Documentation

✅ **GOOD - Comprehensive documentation on public members**:

```csharp
/// <summary>Authenticates user with email and password</summary>
/// <param name="request">Login credentials (email, password)</param>
/// <returns>JWT access token and refresh token for authenticated user</returns>
/// <exception cref="InvalidOperationException">Invalid credentials or user not found</exception>
/// <response code="200">Authentication successful</response>
/// <response code="401">Invalid credentials</response>
/// <response code="429">Rate limited (10 requests/minute)</response>
[HttpPost("login")]
[AllowAnonymous]
[EnableRateLimiting("auth")]
public async Task<ActionResult<LoginResponseDto>> Login([FromBody] LoginRequestDto request) { }
```

❌ **BAD - Missing documentation**:

```csharp
// ❌ No documentation
public async Task<LoginResponseDto> Login(LoginRequestDto request) { }
```

### 7. Nullable Reference Types

✅ **GOOD - Explicit nullability**:

```csharp
public record CreateOrganizationDto(
    [Required] string OrganizationName,
    string? Description,  // ✅ Explicitly nullable
    int? ParentOrgId      // ✅ Explicitly nullable
);

public class Organization
{
    public int OrganizationId { get; set; }

    [Required]
    public string OrganizationName { get; set; } = string.Empty;  // ✅ Non-nullable

    public string? Description { get; set; }  // ✅ Nullable
}
```

❌ **BAD - Ambiguous nullability**:

```csharp
// ❌ Compiler warnings about possible null reference
public string? GetOrgName(Organization org)
{
    return org.Name;  // ⚠️ Could be null, unclear
}
```

### 8. Database Queries - Avoid N+1

✅ **GOOD - Use Include() for relationships**:

```csharp
// ✅ Single query with related data
public async Task<Organization> GetWithEmployeesAsync(int id)
{
    return await _context.Organizations
        .Include(o => o.Employees)
        .FirstOrDefaultAsync(o => o.OrganizationId == id);
}
```

❌ **BAD - N+1 queries**:

```csharp
// ❌ Creates N+1 queries (1 for org, N for each employee)
var orgs = await _context.Organizations.ToListAsync();
foreach (var org in orgs)
{
    var employees = org.Employees;  // ❌ Separate query per org
}
```

### 9. Unit of Work Pattern

✅ **GOOD - Coordinate multiple repositories**:

```csharp
public async Task DeleteAsync(int orgId)
{
    var org = await _unitOfWork.Organizations.GetByIdAsync(orgId);
    var employees = await _unitOfWork.Employees.GetByOrganizationAsync(orgId);

    // Delete in correct order: Employees → Users → Organization
    foreach (var emp in employees)
    {
        await _unitOfWork.Users.DeleteAsync(emp.User);
        await _unitOfWork.Employees.DeleteAsync(emp);
    }

    await _unitOfWork.Organizations.DeleteAsync(org);
    await _unitOfWork.SaveChangesAsync();  // ✅ Single transaction
}
```

### 10. Naming Conventions

| Category         | Convention                        | Example                                          |
| ---------------- | --------------------------------- | ------------------------------------------------ |
| **Classes**      | PascalCase                        | `OrganizationService`, `UserRepository`          |
| **Methods**      | PascalCase, Async ends with Async | `GetByIdAsync`, `CreateAsync`                    |
| **Properties**   | PascalCase                        | `OrganizationId`, `FirstName`                    |
| **Fields**       | \_camelCase (readonly)            | `_orgRepo`, `_unitOfWork`                        |
| **Parameters**   | camelCase                         | `organizationId`, `request`                      |
| **Constants**    | UPPER_CASE or PascalCase          | `MaxUserNameLength`, `DEFAULT_TIMEOUT`           |
| **DTOs**         | `[Action][Entity]Dto`             | `CreateOrganizationDto`, `LoginResponseDto`      |
| **Services**     | `[Entity]Service`                 | `OrganizationService`, `AuthService`             |
| **Repositories** | `[Entity]Repository`              | `OrganizationRepository`, `UserRepository`       |
| **Controllers**  | `[Entity]Controller` (plural)     | `OrganizationsController`, `EmployeesController` |

## When Modifying Backend Code

### Adding a New API Endpoint

1. **Create DTO** in `Application/DTOs/`:

   ```csharp
   public record CreateUserDto(
       [Required][EmailAddress] string Email,
       [Required][MinLength(8)] string Password,
       [Required] string FirstName
   );
   ```

2. **Add Service Method** in `Application/Services/`:

   ```csharp
   public async Task<UserDto> CreateAsync(CreateUserDto dto)
   {
       // Business logic
       var user = new User { Email = dto.Email, ... };
       await _userRepository.AddAsync(user);
       await _unitOfWork.SaveChangesAsync();
       return MapToDto(user);
   }
   ```

3. **Add Repository Method** (if needed):

   ```csharp
   public async Task<User> GetByEmailAsync(string email)
   {
       return await _context.Users
           .FirstOrDefaultAsync(u => u.Email == email);
   }
   ```

4. **Add Controller Endpoint** in `Api/Controllers/`:

   ```csharp
   [HttpPost]
   [Authorize(Roles = "SystemOwner")]
   public async Task<ActionResult<UserDto>> Create([FromBody] CreateUserDto dto)
   {
       if (!ModelState.IsValid)
           return BadRequest(ModelState);

       var user = await _userService.CreateAsync(dto);
       return CreatedAtAction(nameof(GetById), new { id = user.UserId }, user);
   }
   ```

5. **Add XML Documentation** to all public members
6. **Write Tests** in `ElectionVoting.Tests/`
7. **Update API_DOCUMENTATION.md** with new endpoint

### Modifying Database Schema

1. **Update Entity** in `Domain/Entities/`:
2. **Create Migration**: `dotnet ef migrations add {Description}`
3. **Review Migration** SQL for correctness
4. **Apply Migration**: `dotnet ef database update`
5. **Update Repository** if needed for new queries

## Security Checklist

When reviewing backend changes:

- [ ] Passwords hashed (BCrypt), never stored plain text
- [ ] Access tokens in-memory only (JWT, short-lived)
- [ ] Rate limiting on auth endpoints (10 req/min)
- [ ] [Authorize] attribute on protected endpoints
- [ ] Role-based access checks where needed
- [ ] Organization isolation for multi-tenant data
- [ ] Input validation on all DTOs
- [ ] No secrets in code (use configuration)
- [ ] Audit logging on mutations
- [ ] SQL injection prevention (using EF Core parameterized queries)

## Common Patterns

### Service Template

```csharp
public class NewService : INewService
{
    private readonly INewRepository _repo;
    private readonly IUnitOfWork _unitOfWork;

    public NewService(INewRepository repo, IUnitOfWork unitOfWork)
    {
        _repo = repo;
        _unitOfWork = unitOfWork;
    }

    public async Task<NewDto> GetByIdAsync(int id)
    {
        var entity = await _repo.GetByIdAsync(id)
            ?? throw new KeyNotFoundException($"Not found: {id}");
        return MapToDto(entity);
    }

    public async Task<NewDto> CreateAsync(CreateNewDto dto)
    {
        var entity = new New { /* map from dto */ };
        await _repo.AddAsync(entity);
        await _unitOfWork.SaveChangesAsync();
        return MapToDto(entity);
    }

    public async Task DeleteAsync(int id)
    {
        var entity = await _repo.GetByIdAsync(id)
            ?? throw new KeyNotFoundException($"Not found: {id}");
        await _repo.DeleteAsync(entity);
        await _unitOfWork.SaveChangesAsync();
    }
}
```

### Repository Template

```csharp
public class NewRepository : Repository<New>, INewRepository
{
    private readonly ElectionVotingDbContext _context;

    public NewRepository(ElectionVotingDbContext context) : base(context)
    {
        _context = context;
    }

    public async Task<New> GetWithRelatedAsync(int id)
    {
        return await _context.News
            .Include(n => n.RelatedEntity)
            .FirstOrDefaultAsync(n => n.Id == id);
    }
}
```
