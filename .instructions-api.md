## API Philosophy

**Principles**:

1. **REST conventions**: Use correct HTTP methods (GET, POST, PUT, DELETE)
2. **Meaningful status codes**: 200, 201, 400, 401, 403, 404, 422, 429, 500
3. **Consistent error format**: All errors follow same structure
4. **Input validation first**: Validate at boundary (API layer)
5. **Authentication on entry**: [Authorize] on controller or methods
6. **Rate limiting on sensitive**: Auth endpoints (10 req/min)

---

## HTTP Status Codes

| Code    | When to Use                           | Example                                      |
| ------- | ------------------------------------- | -------------------------------------------- |
| **200** | Successful GET or query               | `GET /api/organizations` → List of orgs      |
| **201** | Resource created                      | `POST /api/organizations` → New org created  |
| **204** | Success with no content               | `DELETE /api/organizations/{id}` → Deleted   |
| **400** | Bad request (validation error)        | Missing required field, invalid format       |
| **401** | Unauthorized (needs login)            | Access token missing or invalid              |
| **403** | Forbidden (logged in, no permission)  | Manager accessing different org              |
| **404** | Not found                             | `GET /api/organizations/999` → Doesn't exist |
| **422** | Unprocessable entity (business logic) | Duplicate email, org name exists             |
| **429** | Too many requests (rate limited)      | Auth endpoint hit 10x in 1 min               |
| **500** | Server error                          | Unhandled exception                          |

## Endpoint Design Pattern

### GET (Read - No Auth Required)

```csharp
/// <summary>Retrieves all organizations</summary>
/// <returns>List of organizations</returns>
/// <response code="200">Successfully retrieved organizations</response>
[HttpGet]           // ✅ No path parameter = collection
[Authorize(Roles = "SystemOwner,Manager")]
public async Task<ActionResult<IEnumerable<OrganizationDto>>> GetAll()
{
    var orgs = await _orgService.GetAllAsync();
    return Ok(orgs);  // ✅ 200 with list
}

/// <summary>Retrieves a specific organization by ID</summary>
/// <param name="id">Organization ID</param>
/// <returns>Complete organization details</returns>
/// <response code="200">Organization found</response>
/// <response code="404">Organization not found</response>
[HttpGet("{id}")]   // ✅ Path parameter = single resource
[Authorize(Roles = "SystemOwner,Manager")]
public async Task<ActionResult<OrganizationDto>> GetById(int id)
{
    try
    {
        var org = await _orgService.GetByIdAsync(id);
        return Ok(org);     // ✅ 200 with single org
    }
    catch (KeyNotFoundException)
    {
        return NotFound();  // ✅ 404 - not found
    }
}
```

### POST (Create - Returns Created Resource)

```csharp
/// <summary>Creates a new organization</summary>
/// <param name="dto">Organization creation data</param>
/// <returns>Newly created organization with assigned ID</returns>
/// <response code="201">Organization created successfully</response>
/// <response code="400">Invalid organization data</response>
/// <response code="422">Organization name already exists</response>
[HttpPost]
[Authorize(Roles = "SystemOwner")]
public async Task<ActionResult<OrganizationDto>> Create([FromBody] CreateOrganizationDto dto)
{
    if (!ModelState.IsValid)
    {
        return BadRequest(ModelState);  // ✅ 400 - validation error
    }

    try
    {
        var userId = User.FindFirst("userId")?.Value;
        var org = await _orgService.CreateAsync(dto, int.Parse(userId));

        // ✅ 201 (Created) + Location header + resource
        return CreatedAtAction(nameof(GetById), new { id = org.OrganizationId }, org);
    }
    catch (InvalidOperationException ex)
    {
        return UnprocessableEntity(new { message = ex.Message });  // ✅ 422
    }
}
```

### PUT (Update - Full or Partial)

```csharp
/// <summary>Updates organization information</summary>
/// <param name="id">Organization ID</param>
/// <param name="dto">Updated organization data</param>
/// <returns>Updated organization</returns>
/// <response code="200">Organization updated successfully</response>
/// <response code="400">Invalid update data</response>
/// <response code="404">Organization not found</response>
[HttpPut("{id}")]
[Authorize(Roles = "SystemOwner,Manager")]
public async Task<ActionResult<OrganizationDto>> Update(int id, [FromBody] UpdateOrganizationDto dto)
{
    // ✅ Authorization check for Managers
    var userRole = User.FindFirst(ClaimTypes.Role)?.Value;
    if (userRole == "Manager")
    {
        var orgIdClaim = User.FindFirst("organizationId")?.Value;
        if (!int.TryParse(orgIdClaim, out var userOrgId) || userOrgId != id)
            return Forbid();  // ✅ 403 - Manager accessing different org
    }

    try
    {
        var org = await _orgService.UpdateAsync(id, dto);
        return Ok(org);     // ✅ 200 - updated successfully
    }
    catch (KeyNotFoundException)
    {
        return NotFound();  // ✅ 404 - not found
    }
}
```

### DELETE (Remove Resource)

```csharp
/// <summary>Deletes an organization</summary>
/// <param name="id">Organization ID</param>
/// <response code="204">Organization deleted successfully</response>
/// <response code="401">User not authenticated</response>
/// <response code="404">Organization not found</response>
[HttpDelete("{id}")]
[Authorize(Roles = "SystemOwner")]
public async Task<IActionResult> Delete(int id)
{
    try
    {
        await _orgService.DeleteAsync(id);
        return NoContent();  // ✅ 204 - success, no content
    }
    catch (KeyNotFoundException)
    {
        return NotFound();   // ✅ 404 - not found
    }
}
```

## Authentication & Authorization

### On Controller (applies to all methods)

```csharp
[ApiController]
[Route("api/[controller]")]
[Authorize]  // ✅ All methods require authentication
public class OrganizationsController : ControllerBase { }
```

### On Method (specific roles)

```csharp
[HttpPost]
[Authorize(Roles = "SystemOwner")]  // ✅ Only SystemOwner can create
public async Task<ActionResult<OrganizationDto>> Create(...) { }

[Httpget]
[Authorize(Roles = "SystemOwner,Manager")]  // ✅ Multiple roles
public async Task<ActionResult<IEnumerable<OrganizationDto>>> GetAll() { }

[HttpGet("public")]
[AllowAnonymous]  // ✅ No authentication required
public async Task<ActionResult> GetPublicInfo() { }
```

### Organization Isolation (for Managers)

```csharp
[HttpGet("{id}")]
[Authorize(Roles = "SystemOwner,Manager")]
public async Task<ActionResult<OrganizationDto>> GetById(int id)
{
    // ✅ Managers can only access their own org
    var userRole = User.FindFirst(ClaimTypes.Role)?.Value;
    if (userRole == "Manager")
    {
        var orgIdClaim = User.FindFirst("organizationId")?.Value;
        if (!int.TryParse(orgIdClaim, out var userOrgId) || userOrgId != id)
            return Forbid();  // 403 Forbidden
    }

    return Ok(await _orgService.GetByIdAsync(id));
}
```

## Routing Conventions

| Resource       | GET                       | POST                       | PUT             | DELETE             |
| -------------- | ------------------------- | -------------------------- | --------------- | ------------------ |
| **Collection** | GET /api/orgs             | POST /api/orgs             | N/A             | N/A                |
| **Single**     | GET /api/orgs/1           | N/A                        | PUT /api/orgs/1 | DELETE /api/orgs/1 |
| **Nested**     | GET /api/orgs/1/employees | POST /api/orgs/1/employees | N/A             | N/A                |
| **Action**     | GET /api/orgs/1/summary   | POST /api/data/attendance  | N/A             | N/A                |

```csharp
// ✅ Good routes

[Route("api/[controller]")]
public class OrganizationsController : ControllerBase
{
    [HttpGet]                          // GET /api/organizations
    [HttpGet("{id}")]                  // GET /api/organizations/1
    [HttpPost]                         // POST /api/organizations
    [HttpPut("{id}")]                  // PUT /api/organizations/1
    [HttpDelete("{id}")]               // DELETE /api/organizations/1
}

[Route("api/organizations/{orgId}/employees")]
public class EmployeesController : ControllerBase
{
    [HttpGet]                          // GET /api/organizations/1/employees
    [HttpGet("{empId}")]               // GET /api/organizations/1/employees/5
    [HttpPost]                         // POST /api/organizations/1/employees
    [HttpPut("{empId}")]               // PUT /api/organizations/1/employees/5
    [HttpDelete("{empId}")]            // DELETE /api/organizations/1/employees/5
}
```

## Pagination & Filtering

```csharp
/// <summary>Gets organizations with pagination and filtering</summary>
/// <param name="page">Page number (1-indexed)</param>
/// <param name="pageSize">Items per page (max 100)</param>
/// <param name="search">Search by name</param>
[HttpGet]
public async Task<ActionResult<PagedResult<OrganizationDto>>> GetAll(
    [FromQuery] int page = 1,
    [FromQuery] int pageSize = 10,
    [FromQuery] string search = null)
{
    const int maxPageSize = 100;
    pageSize = Math.Min(pageSize, maxPageSize);

    var (items, total) = await _orgService.GetPagedAsync(page, pageSize, search);

    return Ok(new PagedResult<OrganizationDto>
    {
        Items = items,
        Page = page,
        PageSize = pageSize,
        Total = total,
        TotalPages = (total + pageSize - 1) / pageSize
    });
}

// Client usage: GET /api/organizations?page=2&pageSize=20&search=ElectionCorp
```

## XML Documentation for Swagger

Every public endpoint **must** have XML documentation:

```csharp
/// <summary>
/// Authenticates user with email and password
/// </summary>
/// <remarks>
/// Returns JWT access token for use in Authorization header.
/// Refresh token stored in httpOnly cookie.
/// Rate limited: 10 requests per minute.
/// </remarks>
/// <param name="request">Login credentials (email, password)</param>
/// <returns>JWT tokens and expiration time</returns>
/// <response code="200">Authentication successful, tokens returned</response>
/// <response code="400">Invalid email or password format</response>
/// <response code="401">Email not found or password incorrect</response>
/// <response code="429">Rate limit exceeded (10 req/min)</response>
[HttpPost("login")]
[AllowAnonymous]
[EnableRateLimiting("auth")]
public async Task<ActionResult<LoginResponseDto>> Login([FromBody] LoginRequestDto request) { }
```

## Swagger/OpenAPI Configuration

**Configured in Program.cs**:

```csharp
builder.Services.AddSwaggerGen(c =>
{
    var xmlFile = $"{System.Reflection.Assembly.GetExecutingAssembly().GetName().Name}.xml";
    var xmlPath = Path.Combine(AppContext.BaseDirectory, xmlFile);
    if (File.Exists(xmlPath))
    {
        c.IncludeXmlComments(xmlPath);  // ✅ Auto-loads XML docs
    }
});

app.UseSwagger();
app.UseSwaggerUI();  // ✅ Visit /swagger/ui
```

## Endpoint Checklist

When adding a new endpoint, verify:

- [ ] HTTP method is correct (GET, POST, PUT, DELETE)
- [ ] Route follows conventions (`/api/[controller]` or `/api/parent/{id}/children`)
- [ ] [Authorize] or [AllowAnonymous] specified
- [ ] Request model (DTO) has DataAnnotations validation
- [ ] Response model (DTO) has property documentation
- [ ] All return paths have implicit status codes documented
- [ ] XML documentation includes summary, params, returns, responses
- [ ] Error cases handled: 400, 401, 403, 404, 422
- [ ] Rate limiting added for sensitive operations
- [ ] Meaningful variable names extracted from claims when needed
- [ ] Organization isolation enforced for multi-tenant data
- [ ] Tests written for success and error paths

```csharp

```

### Check Role from Claims

```csharp
var userRole = User.FindFirst(ClaimTypes.Role)?.Value;
if (userRole == "Manager")
{
    // Manager-specific logic
}
```

### Return Created Response

```csharp
return CreatedAtAction(nameof(GetById), new { id = org.OrganizationId }, org);
```
