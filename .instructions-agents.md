## Debugging with AI Agents

### Scenario: Compilation Error

1. **Copy error message into prompt**

   ```
   I'm getting this error:
   [Error] CS1061: 'Employee' does not contain a definition for 'GetVotingHistory'

   This is in EmployeeService.cs line 42.
   The method should exist in the domain layer.
   ```

2. **Copilot suggests**:
   - The missing method in Employee entity
   - The pattern to follow (from `.instructions-domain.md`)
   - Where it belongs (entity vs. specification)

### Scenario: Logic Bug

1. **Describe the flow**

   ```
   Using .instructions-backend.md async patterns:

   When an election ends, I need to:
   1. Mark all open votes as cancelled
   2. Finalize the vote counts
   3. Update the election status
   4. Create audit log

   I'm getting race conditions. How should I handle this atomically?
   ```

2. **Copilot suggests**:
   - Unit of Work pattern
   - Transaction handling
   - Reference to DbContext SaveChanges

### Scenario: Performance Complaint

1. **Provide details**

   ```
   Using .instructions-infrastructure.md:

   The employee list endpoint is slow (2 seconds for 100 employees).
   Currently it does:
   var employees = _context.Employees.ToList();
   foreach(var e in employees) { var org = e.Organization; }

   What's the issue and how do I fix it?
   ```

2. **Copilot identifies**:
   - N+1 query problem
   - References `.Include()` pattern
   - Shows specification solution

## Common AI Pitfalls & Solutions

### ❌ Pitfall: AI suggests non-async code

```
❌ AI suggests:
var employees = _context.Employees.ToList();

✅ Correct:
var employees = await _context.Employees.ToListAsync();
```

**Fix**: Reference `.instructions-backend.md` async section in prompt

### ❌ Pitfall: AI forgets multi-tenant checks

```
❌ AI suggests:
var org = await _orgService.GetByIdAsync(id);
return Ok(org);

✅ Correct includes:
if (userRole == "Manager") {
    var userOrgId = GetOrganizationIdFromClaims();
    if (userOrgId != id) return Forbid();
}
```

**Fix**: Reference security section in prompt

### ❌ Pitfall: AI creates public setters

```
❌ AI suggests:
public string OrganizationName { get; set; }

✅ Correct:
public string OrganizationName { get; private set; }
```

**Fix**: Reference `.instructions-domain.md` immutability section

### ❌ Pitfall: AI forgets error handling

```
❌ AI suggests:
var org = await _orgService.GetByIdAsync(id);
return Ok(org);  // What if not found?

✅ Correct:
try {
    var org = await _orgService.GetByIdAsync(id);
    return Ok(org);
} catch (KeyNotFoundException) {
    return NotFound();
}
```

**Fix**: Reference `.instructions-api.md` error handling section

## Validation Checklist for AI-Generated Code

After asking AI for code, verify:

- [ ] Uses async/await (not sync)
- [ ] Has error handling (try/catch or throws)
- [ ] Includes XML documentation
- [ ] Follows naming conventions
- [ ] No public setters on entities
- [ ] Includes authorization checks [Authorize]
- [ ] Organization isolation enforced (multi-tenant)

- [ ] No circular dependencies
- [ ] Tests can be written easily
