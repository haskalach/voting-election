## Domain Philosophy

**Principles**:

1. **Entities own their rules**: Business logic lives in entities, not services
2. **Immutability-first**: Use records for value objects, sealed on entities
3. **Meaningful names**: Names reflect business, not technical concerns
4. **No circular dependencies**: Layer only flows downward
5. **Complete contracts**: Properties tell the full story of entity state
6. **Validation at boundary**: Constructor and property setters validate

---

## Entity Design Pattern

All entities:

1. Have an **Id** (primary key)
2. Have **timestamps** (CreatedAt, ModifiedAt)
3. Have **audit trail** (CreatedBy, ModifiedBy)
4. **Validate in constructor** (not setter)
5. Use **navigation properties** for relationships
6. Are **sealed** (no inheritance outside domain)

✅ **Good entity structure**:

```csharp
namespace ElectionVoting.Domain.Entities
{
    /// <summary>
    /// Represents an organization in the election voting system.
    /// Organizations are top-level containers for employees, polling stations, and elections.
    /// </summary>
    public sealed class Organization
    {
        // ✅ Constructor validates all required fields
        public Organization(
            string organizationName,
            string managerEmail,
            string adminFirstName,
            string adminLastName,
            int createdBy)
        {
            // ✅ Validate before setting
            if (string.IsNullOrWhiteSpace(organizationName))
                throw new ArgumentException("Organization name is required", nameof(organizationName));

            if (organizationName.Length > 200)
                throw new ArgumentException("Organization name cannot exceed 200 characters", nameof(organizationName));

            if (!IsValidEmail(managerEmail))
                throw new ArgumentException("Invalid email format", nameof(managerEmail));

            if (string.IsNullOrWhiteSpace(adminFirstName) || adminFirstName.Length < 3)
                throw new ArgumentException("Admin first name must be at least 3 characters", nameof(adminFirstName));

            if (string.IsNullOrWhiteSpace(adminLastName) || adminLastName.Length < 3)
                throw new ArgumentException("Admin last name must be at least 3 characters", nameof(adminLastName));

            // ✅ Set after validation passes
            OrganizationName = organizationName;
            ManagerEmail = managerEmail;
            AdminFirstName = adminFirstName;
            AdminLastName = adminLastName;
            CreatedBy = createdBy;
            CreatedAt = DateTime.UtcNow;
            ModifiedAt = DateTime.UtcNow;
            IsDeleted = false;
            TotalEmployeeCount = 0;
        }

        // ✅ Primary key
        public int OrganizationId { get; private set; }

        // ✅ Core properties
        public string OrganizationName { get; private set; }
        public string ManagerEmail { get; private set; }
        public string AdminFirstName { get; private set; }
        public string AdminLastName { get; private set; }

        // ✅ Computed property
        public string AdminFullName => $"{AdminFirstName} {AdminLastName}";

        // ✅ Audit trail
        public int CreatedBy { get; private set; }
        public DateTime CreatedAt { get; private set; }
        public int ModifiedBy { get; private set; }
        public DateTime ModifiedAt { get; private set; }

        // ✅ Soft delete support
        public bool IsDeleted { get; private set; }

        // ✅ Denormalized count for performance
        public int TotalEmployeeCount { get; private set; }

        // ✅ Navigation properties
        public ICollection<Employee> Employees { get; set; } = [];
        public ICollection<PollingStation> PollingStations { get; set; } = [];
        public ICollection<Election> Elections { get; set; } = [];

        // ✅ Business methods modify state
        public void UpdateInfo(string organizationName, string managerEmail, int modifiedBy)
        {
            if (string.IsNullOrWhiteSpace(organizationName))
                throw new ArgumentException("Organization name is required");

            if (!IsValidEmail(managerEmail))
                throw new ArgumentException("Invalid email format");

            OrganizationName = organizationName;
            ManagerEmail = managerEmail;
            ModifiedBy = modifiedBy;
            ModifiedAt = DateTime.UtcNow;
        }

        public void SoftDelete(int modifiedBy)
        {
            IsDeleted = true;
            ModifiedBy = modifiedBy;
            ModifiedAt = DateTime.UtcNow;
        }

        public void Restore(int modifiedBy)
        {
            IsDeleted = false;
            ModifiedBy = modifiedBy;
            ModifiedAt = DateTime.UtcNow;
        }

        // ✅ Domain helper
        private static bool IsValidEmail(string email)
        {
            try
            {
                var addr = new System.Net.Mail.MailAddress(email);
                return addr.Address == email;
            }
            catch
            {
                return false;
            }
        }
    }
}
```

## Value Objects (Immutable, No ID)

Value objects represent concepts without identity:

✅ **Good value object**:

```csharp
/// <summary>
/// Represents an immutable email address with validation.
/// Value objects have no identity and are compared by value.
/// </summary>
public record Email
{
    public string Value { get; }

    public Email(string value)
    {
        if (string.IsNullOrWhiteSpace(value))
            throw new ArgumentException("Email cannot be empty");

        if (!IsValid(value))
            throw new ArgumentException("Invalid email format");

        Value = value;
    }

    private static bool IsValid(string email)
    {
        try
        {
            var addr = new System.Net.Mail.MailAddress(email);
            return addr.Address == email;
        }
        catch
        {
            return false;
        }
    }

    public override string ToString() => Value;

    // ✅ Value objects support implicit conversion
    public static implicit operator string(Email email) => email.Value;
    public static explicit operator Email(string value) => new(value);
}

// Usage:
var email = new Email("user@example.com");  // ✅ Validates on creation
var str = (string)email;                     // ✅ Converts to string
```

## Entity Relationships

### One-to-Many

```csharp
// In Organization (parent)
public ICollection<Employee> Employees { get; set; } = [];

// In Employee (child)
public int OrganizationId { get; private set; }
public Organization Organization { get; set; } = null!;

// ✅ In DbContext
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Organization>()
        .HasMany(o => o.Employees)
        .WithOne(e => e.Organization)
        .HasForeignKey(e => e.OrganizationId)
        .OnDelete(DeleteBehavior.Restrict);  // ✅ Prevent accidental delete
}
```

### Many-to-Many

```csharp
// In Employee
public ICollection<Election> Elections { get; set; } = [];

// In Election
public ICollection<Employee> Employees { get; set; } = [];

// ✅ In DbContext (automatic join table)
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Employee>()
        .HasMany(e => e.Elections)
        .WithMany(el => el.Employees)
        .UsingEntity<EmployeeElection>(
            j => j.HasOne(ee => ee.Election).WithMany(),
            j => j.HasOne(ee => ee.Employee).WithMany());
}
```

## Specifications for Complex Queries

Use specifications to encapsulate complex query logic:

```csharp
/// <summary>Specification for active employees in an organization</summary>
public sealed class ActiveEmployeesByOrganizationSpecification : Specification<Employee>
{
    public ActiveEmployeesByOrganizationSpecification(int organizationId)
    {
        Query
            .Where(e => e.OrganizationId == organizationId && !e.IsDeleted)
            .OrderBy(e => e.LastName)
            .ThenBy(e => e.FirstName);

        // ✅ Include related data to avoid N+1
        Query.Include(e => e.Organization);
    }
}

// Usage in application service:
var spec = new ActiveEmployeesByOrganizationSpecification(orgId);
var employees = await _employeeRepository.ListAsync(spec);
```

## Business Rules (Entity Methods)

Entities should enforce business rules:

```csharp
public sealed class Election
{
    public int ElectionId { get; private set; }
    public string ElectionName { get; private set; }
    public ElectionStatus Status { get; private set; }
    public DateTime StartDate { get; private set; }
    public DateTime EndDate { get; private set; }
    public int OrganizationId { get; private set; }

    // ✅ Business logic in entity, not service
    public bool CanRecordVote()
    {
        return Status == ElectionStatus.Active &&
               DateTime.UtcNow >= StartDate &&
               DateTime.UtcNow <= EndDate;
    }

    public void StartElection(int modifiedBy)
    {
        if (Status != ElectionStatus.Draft)
            throw new InvalidOperationException("Only draft elections can be started");

        if (DateTime.UtcNow < StartDate)
            throw new InvalidOperationException("Cannot start election before start date");

        Status = ElectionStatus.Active;
        ModifiedBy = modifiedBy;
        ModifiedAt = DateTime.UtcNow;
    }

    public void CloseElection(int modifiedBy)
    {
        if (Status != ElectionStatus.Active)
            throw new InvalidOperationException("Only active elections can be closed");

        Status = ElectionStatus.Closed;
        ModifiedBy = modifiedBy;
        ModifiedAt = DateTime.UtcNow;
    }

    public void FinalizeResults(int modifiedBy)
    {
        if (Status != ElectionStatus.Closed)
            throw new InvalidOperationException("Only closed elections can be finalized");

        Status = ElectionStatus.Finalized;
        ModifiedBy = modifiedBy;
        ModifiedAt = DateTime.UtcNow;
    }
}
```

## Soft Delete Pattern

```csharp
public sealed class Employee
{
    public int EmployeeId { get; private set; }
    public bool IsDeleted { get; private set; }  // ✅ Soft delete flag

    public void SoftDelete(int modifiedBy)
    {
        IsDeleted = true;
        ModifiedBy = modifiedBy;
        ModifiedAt = DateTime.UtcNow;
    }

    public void Restore(int modifiedBy)
    {
        IsDeleted = false;
        ModifiedBy = modifiedBy;
        ModifiedAt = DateTime.UtcNow;
    }
}

// ✅ In DbContext, filter soft-deleted by default
protected override void OnModelCreating(ModelBuilder modelBuilder)
{
    modelBuilder.Entity<Employee>()
        .HasQueryFilter(e => !e.IsDeleted);  // Filter automatically
}

// When you NEED to include deleted (e.g., admin restore):
var employees = await _context.Employees
    .IgnoreQueryFilters()
    .Where(e => e.IsDeleted)
    .ToListAsync();
```

## Naming Conventions

| Concept           | Pattern                  | Example                         |
| ----------------- | ------------------------ | ------------------------------- |
| **Entity**        | Singular noun            | `Organization`, `Employee`      |
| **Property**      | PascalCase               | `OrganizationName`, `CreatedAt` |
| **ID property**   | `{EntityName}Id`         | `OrganizationId`, `EmployeeId`  |
| **Collection**    | Plural noun              | `Employees`, `Elections`        |
| **Enum**          | PascalCase               | `UserRole`, `ElectionStatus`    |
| **Enum member**   | PascalCase               | `SystemOwner`, `ActiveElection` |
| **Value object**  | Descriptive noun         | `Email`, `Password`, `Address`  |
| **Specification** | `{Concept}Specification` | `ActiveEmployeesSpecification`  |
| **Exception**     | `{Concept}Exception`     | `InvalidElectionStateException` |

## Checklist for New Entity

When creating a new entity:

- [ ] Has `{Entity}Id` property as primary key
- [ ] Has `CreatedAt` and `ModifiedAt` timestamps
- [ ] Has `CreatedBy` and `ModifiedBy` (user IDs)
- [ ] Constructor validates all required fields
- [ ] Properties have private setters (only getter public)
- [ ] Business methods exist for state transitions
- [ ] XML documentation on class and public members
- [ ] Soft delete support if data is important
- [ ] Navigation properties for relationships defined
- [ ] No circular dependencies upward to Application layer
- [ ] Enum used for fixed set of values
- [ ] Tests verify validation and business rules
- [ ] Related specification created if complex query needed

## Common Mistakes to Avoid

❌ **Business logic in service, entity has no behavior**

```csharp
// ❌ DON'T: Logic in service
public async Task<bool> CanStartElectionAsync(int electionId)
{
    var election = await _electionRepository.GetByIdAsync(electionId);
    return election.Status == ElectionStatus.Draft;
}
```

✅ **DO: Logic in entity**

```csharp
// ✅ DO: Logic in entity
public class Election
{
    public bool CanBeStarted() => Status == ElectionStatus.Draft;
}
```

---

❌ **Setters on entity properties**

```csharp
// ❌ DON'T: Public setter
public string OrganizationName { get; set; }
```

✅ **DO: Private setter**

```csharp
// ✅ DO: Private setter, only constructor/methods modify
public string OrganizationName { get; private set; }
```

---

❌ **Validation in setter**

```csharp
// ❌ DON'T: Late validation
private string _name;
public string Name
{
    get => _name;
    set => _name = string.IsNullOrEmpty(value) ? throw new Exception(...) : value;
}
```

✅ **DO: Constructor validation**

```csharp
// ✅ DO: Validate early in constructor
public Entity(string name)
{
    if (string.IsNullOrEmpty(name)) throw new ArgumentException(...);
    _name = name;
}
```
