# Survey Template

**Monthly developer survey for measuring adoption, productivity, and satisfaction.**

---

## Survey Overview

### Purpose

Collect quantitative and qualitative feedback on AI-assisted development tools.

### Schedule

- **Frequency**: Monthly (same week each month)
- **Day**: First Monday of month
- **Deadline**: Friday 5:00 PM (4 business days)
- **Audience**: All developers (mandatory)
- **Duration**: 3 minutes
- **Format**: Google Form or online survey tool

### Target Response Rate

✅ **100%** (Everyone should respond)

### Question 5: Time Saved

```
🔷 Question 5: Roughly how many hours per week did AI tools save you?

Hours saved per week: [____] (0-10)

"I estimate tools saved me about ___ hours this week"

Why ask: Quantify productivity gain
How analyze:
  - Avg hours saved
  - If low (<1 hr): Tools not delivering
  - If high (5+ hrs): Tools critical for work
  - Multiply across team

Example calculation:
10 developers × 2.5 hrs/week = 25 hrs/week
25 hrs × $150/hr = $3,750/week benefit
$3,750 × 52 = $195,000 annual benefit
```

### Question 8: Net Promoter Score (NPS)

```
🔷 Question 8: How likely would you recommend these AI tools to a
colleague? (0-10 scale)

0 = Would not recommend at all
5 = Neutral
10 = Would definitely recommend

[____]

Why ask: Industry-standard satisfaction metric
How analyze:
  Promoters (9-10): Likely to recommend
  Passives (7-8): Neutral
  Detractors (0-6): Unlikely to recommend

  NPS = (Promoters - Detractors) / Total × 100

  Target: NPS > 30 (good), > 50 (very good)
```

### Question 9: Learning Curve

```
🔷 Question 9: When did you feel confident using the tools?

[ ] Still learning
[ ] <1 hour of practice
[ ] 1-4 hours of practice

[ ] Not applicable (didn't try)

Why ask: Identify if tools are too complex
How analyze:
  - % learning <4 hours (target: >80%)
  - % still learning (target: <10%)
  - If high "still learning": tools too complex

## Section D: Problems & Improvements

### Question 10: Top Challenge / Frustration

```
🔷 Question 10: What's been your biggest frustration? (Select one)

[ ] Hard to learn / steep learning curve
[ ] Doesn't fit my workflow / feels tacked on
[ ] Results not reliable / inconsistent quality
[ ] Too slow / adds overhead
[ ] Not enough examples / unclear how to use
[ ] Integration not smooth / IDE friction
[ ] No problem / everything great!

Why ask: Identify main blockers to adoption
How analyze:
  - Most common frustration = highest priority to fix
  - "No problem" = tool is working well

```

### Question 11: Key Improvement

```
🔷 Question 11: What ONE thing would make these tools 10x better
for you?

[Large text box - encourage specific ideas]

Example responses might be:
- "Better templates for my backend services"
- "Video tutorials instead of reading docs"
- "Easier integration with our code review process"

  - "Prefer manual" = might not be the right fit

## Survey Cadence & Evolution

### Months 1-3 (Pilot Phase)

15-question survey, monthly  
Clear metrics on adoption, productivity, satisfaction

### Months 4-6 (Growth Phase)

Same 15-question survey + 3 additional questions:

- More detailed satisfaction breakdowns
- Engagement vs adoption detail
- Specific tool depth (Copilot vs prompts)

### Months 7-12 (Optimization Phase)

Streamlined 12-question survey:

- Drop lowest-value questions
- Focus on KPIs that matter most
- Add forward-looking questions
