# Retrospective Template

**Monthly team retrospective meeting structure and facilitation guide.**

---

## Retrospective Overview

**When**: Last Friday of each month, 30 minutes  
**Format**: In-person or video call (all team members)  
**Facilitator**: Project Lead  
**Output**: Action items for next iteration

---

## Pre-Retrospective Setup (Day Before)

### Facilitator Preparation Checklist

- [ ] Send calendar invite (24 hours before)
- [ ] Prepare dashboard with latest metrics
- [ ] Compile survey results summary
- [ ] List improvements from past month
- [ ] Identify trending concerns/wins
- [ ] Prepare slide deck or whiteboard
- [ ] Test video/recording setup
- [ ] Gather notes from 1-on-1s

### Pre-Meeting Communication

**Slack announce (Thursday):**

```
📊 Monthly Retrospective Tomorrow
When: Friday 2:00 PM PT
Where: [Zoom link]
Duration: 30 min
What to bring: Your feedback!

Quick preview:
✅ April improvement results
📈 Metrics dashboard update
💭 Team wins to celebrate
🎯 Planning for May
```

## Retrospective Agenda (30 minutes)

### 1. Welcome & Context (2 minutes)

**Facilitator**:

```
"Welcome everyone! In April, we deployed AI tools and started
collecting feedback. Today we're going to review what worked,
what we learned, and what to do next.

Goals for today:
1. Celebrate wins
2. Understand challenges
3. Plan May improvements
4. Walk out excited 🚀

Let's go!"
```

### 2. Metrics Presentation (5 minutes)

**Show Dashboard**:

```
April 2026 Results:
├─ NPS Score: 35 (target: 30+) ✅ Exceeded!
├─ Adoption Rate: 67% (target: 50+) ✅ Exceeded!
├─ Team Satisfaction: 3.8/5 (target: 3.5+) ✅ Exceeded!
├─ Code Review Time: 6 min (target: 5 min by Month 3) 📈 On track
├─ Test Writing Time: 12 min (target: 10 min by Month 3) 📈 On track
├─ Sprint Velocity: +7% (target: +15% by Month 3) 📈 On track
├─ Test Coverage: 100% (target: maintain) ✅ Maintained
├─ Security Issues: 0 (target: 0) ✅ Maintained
└─ Time Saved: 2.65 hr/week per developer
```

**Talking points**:

```
"We're exceeding adoption and satisfaction targets already!
The velocity improvements are starting to show.
You're saving ~2.6 hours per week thanks to tools.
Let's keep this momentum going."
```

### 3. Wins & Celebrations (5 minutes)

**Ask the Team**:

```
"What went really well in April? What are you proud of?"

Guide them with examples:
- Did code reviews feel smoother?
- Did tests feel easier to write?
- Did documentation improve?
- Did you learn new tools quickly?
- Did anyone have an 'aha!' moment?
```

**Record on Visible Board**:

```
🎉 APRIL WINS:

✅ 67% adoption - better than expected
✅ Fast code reviews - 6 min becomes standard
✅ Comprehensive tests - 100% coverage maintained
✅ Better docs - Swagger now fully populated
✅ Everyone participated - no one left behind
```

**Celebrate Success**:

```
"Amazing work everyone! Let's give ourselves some applause! 👏
April was about launching, and you crushed it.
The fact that 67% are actively using tools is huge."
```

### 4. Challenges & Learnings (8 minutes)

**Ask the Team**:

```
"What was challenging? What didn't go as expected?
Where did we struggle? What can we improve?"

Be specific:
- "Did any tools feel hard to learn?"
- "Were there frustrating moments?"
- "Did anything not work as expected?"
- "What would have helped?"
```

**Document on Board**:

```
⚠️ CHALLENGES:

❌ Learning curve - 4 people found onboarding steep
   → Action: Better examples and videos needed

❌ Some templates unclear - 3 people wanted more guidance
   → Action: Backend-specific templates

❌ Workflow integration - 2 people said tools don't fit
   → Action: Understand their workflow better

❌ Error messages confusing - 1 person frustrated
   → Action: Improve error handling in prompts

⚡ LEARNINGS:

📌 Backend developers especially want templates for services
📌 Video tutorials are more helpful than written docs
📌 People learn fastest by doing, not reading
📌 Early wins build confidence fast
```

### 5. Team Improvement Ideas (5 minutes)

**Ask (Brainstorm Mode)**:

```
"If you could improve one thing for May, what would it be?
No idea is too big or small.
No judgment - just ideas."

Examples:
- Better templates for my domain
- Video tutorials
- Faster integration with my IDE
- More examples
- Easier error recovery
```

**Vote on Priorities**:

```
"Let's narrow down. Which 3 would help you most?"

Show 3 voting categories:
🔴 URGENT (blocking me)
🟡 IMPORTANT (would help a lot)
🟢 NICE-TO-HAVE (would be nice)

Tally votes for each improvement.
Top 3 become May work items.
```

### 6. May Plan & Ownership (4 minutes)

**Facilitator Proposes**:

```
"Based on your feedback, here's what we'll do in May:

🔴 Priority #1: Backend Templates
   Owner: Dev#1 | Target: May 31
   Why: 3 people asked for this
   Impact: Better productivity for backend team

🟡 Priority #2: Video Tutorials
   Owner: Dev#2 | Target: May 15
   Why: Videos help learning
   Impact: Easier onboarding

🟡 Priority #3: Onboarding Guide Update
   Owner: Lead | Target: May 10
   Why: Making setup faster for new people
   Impact: Faster ramp-up time

Questions? Concerns? Any conflicts?"
```

**Confirm Ownership**:

```
"Dev#1: Can you own backend templates?"
Dev#1: "Yes, I can do that"

"Dev#2: Can you own videos?"
Dev#2: "Yep, I'll get those recorded"

"Lead: Can you update the onboarding?"
Lead: "Already working on it"

Great! Confirmed for May!"
```

### 7. Closing & Commitment (1 minute)

**Facilitator**:

```
"Before we finish, let's commit to moving forward.

This month:
- April: We launched and exceeded expectations
- May: We'll build on that with improvements you requested
- June: We'll measure impact and celebrate results

I'm excited about the momentum. Let's keep crushing it!

Any last questions? Concerns? Ideas?"

[Brief Q&A]

"Thanks everyone. See you next month!
Great work in April! 🚀"
```

## Alternative Formats (If In-Person Not Possible)

### Remote-First Retrospective

**Format**: Async (Google Form) + Sync (15-min standup)

**Async Survey** (48 hours before):

```
Google Form with 5 questions:
1. What went well in April? (text)
2. What was challenging? (text)
3. Rate satisfaction 1-5 (scale)
4. What should we improve? (text)
5. Would you recommend tools? 0-10 (scale)

Facilitator: Compile responses before standup
```

**Sync Standup** (15 minutes):

```
Facilitator shares: "Here's what 15 people said..."
Team: Discuss & plan top 3 improvements
Vote: Any strong disagreements?
Commit: "Here's the plan for May"
```

## Retrospective Facilitation Tips

### If Discussion Gets Off-Track

```
"I love that idea, but let's focus on how tools are working.
Can we capture that and discuss separately?"

(Redirect back to agenda, but don't dismiss)
```

### If Someone Seems Frustrated

```
"I noticed you seemed frustrated. Help me understand..."

(Listen actively, take notes, validate)

"That's really valuable feedback. Let's make sure this
becomes a May improvement."
```

### If Adoption Is Low

```
"I notice 33% isn't using tools yet. That's important data.

Let's dig in:
- Are they not interested?
- Do they not understand?
- Do the tools not fit their work?

Who can help us understand better?"

(Create action item to investigate)
```

### If Metrics Are Down

```
"Hmm, code review time went up this month (8 min vs 6).
Let's understand why.

Did something change? Were we measuring differently?
Or did we have more complex PRs?"

(Don't assume failure, investigate)
```

## Success Criteria for Retrospectives

✅ **Attendance**: 100% of team present  
✅ **Engagement**: Everyone speaks at least once  
✅ **Positivity**: Balance of wins and challenges  
✅ **Actionability**: Clear improvements identified  
✅ **Ownership**: Every item has a clear owner  
✅ **Timeliness**: Actions scheduled with deadlines  
✅ **Impact**: Previous month improvements visible  
✅ **Continuation**: Team sees feedback → action → impact cycle
